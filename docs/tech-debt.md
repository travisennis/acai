# Technical Debt

This document tracks known technical debt in the acai project.

## Current Debt

## Categories

### Code Debt
- Refactoring opportunities
- Code smells or complexity issues
- Legacy patterns to modernize

### Documentation Debt
- Missing or outdated docs
- Incomplete specifications
- Undocumented edge cases

### Testing Debt
- Missing test coverage
- Flaky tests
- Slow test suites

### Infrastructure Debt
- Build system issues
- CI/CD improvements
- Dependency updates

## Process

When adding new debt:
1. Describe the issue and its impact
2. Note when it was introduced (if known)
3. Estimate effort to resolve
4. Document any blockers or dependencies

When resolving debt:
1. Update this document with resolution date
2. Link to the PR or commit that addressed it
3. Move to "Resolved Debt" section below
